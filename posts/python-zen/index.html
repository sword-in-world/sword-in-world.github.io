Python 之禅

发表于 2026-07-18 · 分类：Python · 标签：#Python, #编程哲学, #学习笔记

什么是 Python 之禅

Python 之禅（The Zen of Python）是 Tim Peters 在 1999 年写下的 19 条格言，定义了 Python 的设计哲学和编程美学。

在 Python 中输入 import this 就能看到全文：

python 
import this

运行后会输出以下内容：

text 
The Zen of Python, by Tim Peters

Beautiful is better than ugly.
Explicit is better than implicit.
Simple is better than complex.
Complex is better than complicated.
Flat is better than nested.
Sparse is better than dense.
Readability counts.
Special cases aren't special enough to break the rules.
Although practicality beats purity.
Errors should never pass silently.
Unless explicitly silenced.
In the face of ambiguity, refuse the temptation to guess.
There should be one-- and preferably only one --obvious way to do it.
Although that way may not be obvious at first unless you're Dutch.
Now is better than never.
Although never is often better than *right* now.
If the implementation is hard to explain, it's a bad idea.
If the implementation is easy to explain, it may be a good idea.
Namespaces are one honking great idea -- let's do more of those!

逐条解读

1. 优美胜于丑陋

python 
# 丑陋
a,b,c=1,2,3;x=a if a>b else b;print(x)

# 优美
max_value = max(a, b)
print(max_value)

代码是写给人看的，顺便能让机器执行。

2. 显式胜于隐式

python 
# 隐式（不好）
from os import *

# 显式（好）
import os
os.path.join('folder', 'file.txt')

不要用 import *，明确知道你用了什么。

3. 简单胜于复杂

python 
# 复杂
result = []
for i in range(10):
    if i % 2 == 0:
        result.append(i * 2)

# 简单
result = [i * 2 for i in range(10) if i % 2 == 0]

列表推导式一行搞定，清晰易读。

4. 可读性很重要

python 
# 不好读
def f(x,y):return x+y if x>0 and y>0 else 0

# 好读
def add_positive(a, b):
    """两个正数相加，否则返回 0"""
    if a > 0 and b > 0:
        return a + b
    return 0

变量名要有意义，函数要加文档字符串。

5. 错误不应被静默忽略

python 
# 不好
try:
    result = 10 / 0
except:
    pass  # 出错了但你根本不知道

# 好
try:
    result = 10 / 0
except ZeroDivisionError as e:
    print(f"除零错误: {e}")
    result = 0

6. 面对歧义，拒绝猜测

python 
# 模糊
def process(data):
    return data.strip().lower().split()

# 明确
def split_words(text: str) -> list:
    """将文本分割为小写单词列表"""
    return text.strip().lower().split()

用类型注解和文档说明你的意图。

7. 做比不做好，但盲目不如不做

python 
# 想好了再做
goal = "掌握 Python 基础"
progress = 0
steps = ["变量", "控制流", "函数", "列表", "字典", "文件"]
for step in steps:
    progress += 1
    print(f"已完成第 {progress} 步：{step}")
print(f"今天又进步了一点")

# 不要盲目开始一个没想清楚的计划

实践：用 Python 之禅写代码

学了 Python 基础语法，顺手写一段日常代码：

python 
# 用 f-string 做日报
name = "Python Learner"
today = "D1"
topics = ["变量", "f-string", "运算符", "类型转换"]
hours = 2.5

print(f"==={today} 学习打卡===")
print(f"内容：{' / '.join(topics)}")
print(f"时长：{hours} 小时")
print(f"完成度：{len(topics)} 个模块 ✅")

# 简单计算
daily_goal = 4  # 每日目标小时
completion = hours / daily_goal * 100
print(f"完成率：{completion:.0f}%")

输出：

text 
===D1 学习打卡===
内容：变量 / f-string / 运算符 / 类型转换
时长：2.5 小时
完成度：4 个模块 ✅
完成率：62%

总结

原则 一句话
优美 代码是写给人看的
显式 明确比隐晦好
简单 能简单就别复杂
可读性 变量名要有意义
错误处理 别吞掉异常
反对猜测 类型注解+文档

Now is better than never. — 现在开始，胜过永不开始。
